Week 2

My grandfather always liked to talk about the 6 P's. Proper Prior Planning Prevents Poor Performance.

In this case, although it doesn't have the same ring to it. Proper Prior Pseudo-coding Prevents Poor Performance.

The importance of having a plan before you start to build your projects is one of my biggest takeaways from my Javascript projects. Not only would I have been able to write better code, I probably could have shaved hours off the development process by simply spending more time planning. Preparing what you are going to code is just as important as coding itself.

Good code should be well planned. It should be dry and readable by other developers who have no background on the project.

The code should be written in a way that is efficient regarding the CPU's processing power. Anything being repeated should be condensed into functions.

An easy one is writing good code.

But also being able to listen and understand what the needs are for the project. Proper planning and using other developers as sounding boards for ideas. Don't go at it alone. I think that something that will help me is consistently asking my peers, "What do you think?".

Week 5

I wouldn't call CLIs scary, but would actually describe them as intimidating due to the amount of control they give you over system files.

I am old enough to remember using 3.5 and what I believe were the 5 1/4 inch floppy disks. During elementary school we would have to run some sort of .exe file from the command line to initialize the games on the floppy disk. Mavis Beacon, Oregon Trail or Commander Keen anyone?

I've also used them more recently in my previous job before the bootcamp to to move large amounts of files. However, these were commands (.bat) built by others to accomplish these tasks.

They are basically the text based commands of what you accomplish on a GUI. Instead of right-clicking and saying Create New Folder, we are typing mkdir and the new folder name. However, it does require a bit more thinking about where you are in the file structure.

Right now with React, it would be to initialize a React project but also import any of the common components and node modules that I use in most projects.
